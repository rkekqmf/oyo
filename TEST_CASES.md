# OYO Test Cases

현재 구현된 기능을 기준으로 수동 테스트 케이스를 정리한 문서입니다.
새 기능 추가 시 이 문서에 같은 형식으로 확장해 주세요.

## 공통 사전 조건

- backend 실행: `cd backend && npm run dev`
- frontend 실행: `cd frontend && npm run dev`
- 프론트 접속: `http://localhost:5173`
- 백엔드 헬스체크: `http://127.0.0.1:8787/api/health` -> `ok: true`

---

## TC-001 검색 기본 동작

- 목적: 캐릭터 검색 API 호출 및 결과 렌더 확인
- 절차:
  1. 검색창에 캐릭터명 입력
  2. `조회` 클릭
- 기대 결과:
  - 로딩 상태 표시
  - 검색 결과 카드 1개 이상 렌더
  - 오류 없이 화면 유지

## TC-002 검색 예외 처리 (빈 입력)

- 목적: 잘못된 입력 처리 확인
- 절차:
  1. 검색창 비운 상태
  2. `조회` 클릭
- 기대 결과:
  - API 호출 발생하지 않음
  - 앱 오류 없음

## TC-003 상세 모달 열기/닫기

- 목적: 카드 클릭 후 상세 조회 정상 동작 확인
- 절차:
  1. 검색 결과 카드 클릭
  2. 모달 노출 확인
  3. `닫기` 버튼 클릭
- 기대 결과:
  - 상세 API 호출 1회 발생
  - 모달 내 장비/각인/보석 탭 노출
  - 닫기 시 모달 제거

## TC-004 상세 탭 전환

- 목적: 모달 내 탭 전환 UI/데이터 확인
- 절차:
  1. 모달에서 `장비` 확인
  2. `각인` 클릭
  3. `보석` 클릭
- 기대 결과:
  - 탭별 데이터 또는 빈 상태 문구 정상 노출
  - 탭 active 스타일 정상 적용

## TC-005 최근 검색 저장

- 목적: 검색 성공 시 최근 검색 목록 반영 확인
- 절차:
  1. 서로 다른 캐릭터 2~3개 검색
  2. 최근 검색 섹션 확인
- 기대 결과:
  - 최신 검색어가 상단
  - 중복 검색 시 중복 없이 맨 위로 이동

## TC-006 최근 검색 재사용/삭제

- 목적: 최근 검색 동작 확인
- 절차:
  1. 최근 검색어 버튼 클릭
  2. `전체 삭제` 클릭
- 기대 결과:
  - 클릭한 검색어로 즉시 재조회
  - 삭제 후 최근 검색 목록 비어 있음

## TC-007 즐겨찾기 추가/해제

- 목적: 즐겨찾기 토글 및 상태 반영 확인
- 절차:
  1. 결과 카드의 `☆` 클릭
  2. 즐겨찾기 섹션 노출 확인
  3. 같은 카드에서 `★` 클릭
- 기대 결과:
  - 추가 시 즐겨찾기 섹션에 카드 표시
  - 해제 시 목록에서 제거

## TC-008 새로고침 복원

- 목적: localStorage 복원 동작 확인
- 절차:
  1. 최근 검색/즐겨찾기 데이터 만든 상태에서 새로고침
- 기대 결과:
  - 최근 검색/즐겨찾기가 유지되어 표시됨

## TC-009 보스 준비도 체크 기본

- 목적: 보스 준비도 점검 흐름 확인
- 절차:
  1. 보스 선택
  2. 캐릭터 선택
  3. `준비도 점검` 클릭
- 기대 결과:
  - 점검 중 로딩 텍스트 표시
  - 결과 요약(`x/y 통과`) 표시
  - 항목별 통과/미달 배지 표시

## TC-010 보스 준비도 예외 (선택 누락)

- 목적: 입력 누락 시 검증 메시지 확인
- 절차:
  1. 캐릭터 미선택 상태에서 `준비도 점검`
- 기대 결과:
  - 에러 메시지: 보스/캐릭터 선택 안내
  - 앱 크래시 없음

## TC-011 보스 준비도 데이터 없음 상태

- 목적: 캐릭터 데이터 없을 때 안내 노출 확인
- 절차:
  1. 검색/즐겨찾기 데이터가 없는 상태로 진입
- 기대 결과:
  - "먼저 캐릭터를 검색..." 안내 문구 표시
  - 점검 버튼 비활성화

## TC-012 API 응답 실패 처리

- 목적: 네트워크/API 실패 시 사용자 에러 노출 확인
- 절차:
  1. backend 중지
  2. frontend에서 검색 또는 보스 점검 시도
- 기대 결과:
  - 사용자 친화적 에러 메시지 노출
  - 무한 로딩/화면 깨짐 없음

## TC-013 배포 환경 점검

- 목적: 실서버 Pages + Workers 연결 검증
- 절차:
  1. 배포 URL 접속
  2. 검색/상세/보스 준비도 각 1회 실행
  3. 브라우저 Network 확인
- 기대 결과:
  - API 호출이 배포 Worker URL로 정상 전송
  - 4xx/5xx 비정상 연속 발생 없음

## TC-014 캐릭터 비교 기본

- 목적: 2명 이상 선택 후 비교 테이블 생성 확인
- 절차:
  1. 캐릭터 2명 이상 검색/즐겨찾기 준비
  2. 캐릭터 비교 섹션에서 2명 체크
  3. `비교 실행` 클릭
- 기대 결과:
  - 비교 중 로딩 문구 표시
  - 비교 테이블 렌더
  - 아이템레벨/보석/각인 컬럼 표시

## TC-015 캐릭터 비교 선택 제한

- 목적: 비교 인원 제한 동작 확인
- 절차:
  1. 캐릭터 4명 이상이 있는 상태에서 4번째 선택 시도
- 기대 결과:
  - "최대 3명..." 에러 메시지 노출
  - 기존 3명 선택 유지

## TC-016 캐릭터 비교 검증 (최소 2명)

- 목적: 비교 실행 최소 인원 검증 확인
- 절차:
  1. 1명만 선택한 상태에서 `비교 실행`
- 기대 결과:
  - "최소 2명..." 에러 메시지 노출
  - 앱 크래시 없음

## TC-017 경매장 자동 수집 확인 (cron)

- 목적: 배포 환경에서 4시간 주기 자동 수집 확인
- 절차:
  1. `AUCTION_WATCH_ITEMS`에 아이템 등록
  2. cron 주기(4시간) 이후 데이터 조회
- 기대 결과:
  - 등록 아이템의 데이터 포인트 누적
  - 기간 조회 시 차트 렌더 가능

## TC-018 경매장 기간별 조회

- 목적: 30/90/365일 히스토리 조회 확인
- 절차:
  1. 기간 선택 변경
  2. `추이 조회` 클릭
- 기대 결과:
  - 기간별 데이터 조회
  - 라인 차트 렌더
  - 시작/끝 날짜 라벨 노출

## TC-019 경매장 빈 데이터 안내

- 목적: 수집 전 상태 UI 확인
- 절차:
  1. 데이터가 없는 신규 아이템명 입력
  2. `추이 조회` 클릭
- 기대 결과:
  - 안내 문구 노출
  - 앱 크래시 없음
