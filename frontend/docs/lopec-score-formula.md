# 로펙 점수 계산식 정리

## 목적
- 특정 캐릭터만 정확히 맞추는 하드코딩형 다항식(과적합)을 피하고,
- 새로운 캐릭터에서도 점수가 과도하게 튀지 않도록 **저차 일반화 모델**을 사용합니다.

## 현재 적용 식
- 코드 위치: `src/utils/combatPower.js`
- 함수: `calibrateLopecLikeScore(powerRaw, gemAverageLevel, itemLevel)`

```
score = c0 + c1*raw + c2*gem + c3*itemLevel + c4*(raw*itemLevel)
```

계수:
- `c0 = 222941.07801556587`
- `c1 = -0.25743857692759775`
- `c2 = -353.05095523557975`
- `c3 = -130.64954627200495`
- `c4 = 0.00016611100669061685`

후처리:
- `score = max(0, round(score, 2))`

## 변수 의미
- `raw`: 내부 전투력 계산 결과  
  (`공격력 × 스탯보정 × 각인보정 × 보석보정 × 아크패시브보정 × 추가보정`)
- `gem`: 평균 보석 레벨
- `itemLevel`: 평균 아이템 레벨(숫자 파싱)

- 이전 13항 다항식은 앵커를 거의 0오차로 맞췄지만, 미등록 캐릭터에서 점수가 크게 깨졌습니다.

## 다음 튜닝 포인트(검토용)
아래 항목은 필요시 추가/제거 후보입니다.
- 추가 후보: `raw*gem`, `gem*itemLevel`
- 제거 후보: `raw*itemLevel` (영향이 과하면 제거)
- 입력 확장 후보: 직업별 더미 변수(클래스 보정), 각인/아크패시브 요약 지표
- 학습 방식: 단순 최소제곱 → 릿지(정규화)로 계수 폭 제한
